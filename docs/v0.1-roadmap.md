# v0.1 — Roadmap

Goal: rubino usable as a backend by an HTTP client (client-ui being the reference consumer), with a clean REST surface and OAuth provider connectors.

## Definition of done

- HTTP gateway under `/v1`, Bearer-authed, all input dry-schema validated, typed errors mapped to HTTP statuses.
- All endpoints in `docs/api/v1.md` implemented and contract-tested.
- OAuth connectors for Github + Google, with encrypted token storage and a documented setup guide.
- Cron jobs schedulable with cron syntax, webhook delivery on completion.
- Structured JSON logging, metrics endpoint, health endpoint.
- README explains install + run in <5 minutes.
- `bundle exec rspec` green. `rubocop` clean.

**Total: ~3-4 weeks wall-clock, one person.**

Order matters: 1 unlocks all others. 2/3/4 are parallelizable after 1. 5 and 6 depend only on 1. 7/8/9/10 layer in continuously.

## What v0.1 does NOT include

- Browser automation
- Embeddings / semantic search
- Multi-workspace
- Web search built-in
- Dashboard / themes / plugin hub
- OAuth providers beyond Github + Google (Slack/Notion/Atlassian are postponed but the registry is extensible)
- WebSocket / PTY endpoints
- legacy-style profiles (one workspace per instance, period)

These are not "next milestones" — they are explicitly deferred until concrete use cases demand them.

## v0.2 candidate list (not commitments)

- Postgres backend option
